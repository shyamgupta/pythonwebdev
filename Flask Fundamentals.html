2.0 Connecting to Database using SQLAlchemy

Lines 5-11 in the below code are being used to connect to our restaurant database.
  • In line 7 we are importing the Base, Restaurant, MenuItem classes (which represents our database tables) from the db_setup.py file.
  • In line 8, we are specifying the database name we are connecting to.

3.0 Routes

We can make the Flask routes dynamic by adding variables to it as shown below:
/path/<type:variable_name>/path
Here, the type can be an integer, string or another path. For example, in the below code, we have created a new route usiing which we can pull restaurant details by providing the restaurant id in the URL
@app.route('/restaurants/<int:restaurant_id>/')
def show_restaurant(restaurant_id):
restaurant = session.query(Restaurant).filter_by(id = restaurant_id).one()
menu_items = session.query(MenuItem).filter_by(restaurant_id= restaurant_id)
output = "Restaurant Name is {}. Below are its menu items".format(restaurant.name)
output += '<br>'
for item in menu_items:
output += item.name + ', ' + item.course + ', ' + item.description + ', ' + item.price
output += '<br>'
return output

4.0 Rendering Templates

Flask will look for HTML templates in a directory called ‘templates’ in our projects root directory. We can use the render_template() function to render the HTML files:
  1. The first argument is the name of the file we want to render
  2. Remaining arguments must be key-value pairs.
render_template('file_name.html', key=value, key=value..)
We can then use Flask’s Jinja2 template engine and execute Python code in the templates using {{ ..}} and {% .. %}
#server.py file
@app.route('/)
def hello_world():
return render_template('index.html',first_name='John',last_name='Doe', times=5)
<!--/project_name/templates/index.html-->
<p>My name is {{first_name}} {{last_name}}</p>
<p>Printing my name {{times}} times</p>
{% for i in range(0,times): %}
<p>{{first_name}}</p>
{% endfor %}

5.0 Static Files

To serve static files (javascript, css, images..) we need to create a static folder in our projects root directory. Within this folder we can create separate folders named css, js, img to serve Javascript, CSS, and Images. We can then link these files in our HTML templates as shown below.
<!-- linking a css style sheet -->
<link rel="stylesheet" type="text/css" href="{{ url_for('static', filename='css/my_style_sheet.css') }}">
<!-- linking a javascript file -->
<script type="text/javascript" src="{{ url_for('static', filename='js/my_script.js') }}"></script>
<!-- linking an image -->
<img src="{{ url_for('static', filename='img/my_img.png') }}">

6.0 URL Building: url_for() and redirect()

We can create URL’s based on the functions they execute - this is useful when we want to create links to other pages inside our application.To build a URL for a function, we can use the following function:
import url_for,redirect
return redirect(url_for(function_name, key=value, key=value..))
url_for( ) function returns the URL - for example, 
url_for(‘hello_world’) from section 1 will return ‘/‘. 
url_for(’show_restaurants’, restaurant_id=2) will return /restaurants/2
This can now be used within the redirect( ) function and we can avoid hardcoding URLs. It’s a best practice to use url_for( ) within redirect( )

7.0 Handling POST Requests

To handle POST requests, we will need to do the following:

Client Side HTML

  • The action attribute of the form tag should be set to the route where the form data will be sent once the form is submitted.
  • The method attribute of the form should be set to POST.
  • The form elements (like input, select, etc) should have a name attribute - the value of this attribute will be transferred to the server when the form is submitted.

Server Side: Handling POST data on the server

  • Import the request and redirect modules in the server file.
  • Route name should be the same as the one specified in the action attribute of the HTML form.
  • We also need to specify the methods=['POST'] argument in the decorator.
  • We can access the form data through the request.form dictionary by providing the name of the input as the key. Note: Type of the submitted form data is always a String.
  • Once the function associated with the decorator is processed we can redirect to the root route by using redirect('/'). Note: Its a best practice to always redirect after handling POST data to avoid data being handled more than once!

<input type='submit' value='Create User'>
</form>

8.0 Sessions

Sessions are server side files that contain user infromation, while Cookies are client side files that also contain user information. Since sessions are stored on the server, clients do not have access to the information stored about them. Sessions work like atoken, allowing access and passing information while the client browser session is open. Once the browser is closed, we lose the session.
We use sessions to store user information in the form of Python dictionary with key-value pairs. Using sessions involves the following steps:
  1. Import ‘session’ from flask.
  2. Create a secret key for our application.
  3. We can now use the session dictionary to store user inputs received from the form, session[‘name’] = request.form[‘first’]
When you want to access and modify data over multiple redirects use session.
Also, we can access the information stored in the session dictionary directly in out HTML templates, without even passing them in the render_template( ) function.
#server.py file
from flask import Flask, render_template, request,redirect, session
app = Flask(__name__)
app.secret_key = 'My secret key'
@app.route('/users',methods=['POST'])
def create_user():
session['first_name'] = request.form['first']
session['last_name'] = request.form['last']
return redirect('/show')
@app.route('/show')
def show_user():
return render_template('user.html',first=session['first_name'],last=session['last_name'])
<!--/project_folder/templates/user.html-->
<p>{{first}}</p>
<p>{{last}}</p>

9.0 Hidden Inputs

Hidden input fields are form fields that are hidden from the user. Hidden input is used, along with other input elements, to transfer information between different pages. A hidden input is just an ordinary input element but has a type of "hidden”. One of the ways we can use hidden inputs is a page which has two forms, and we can use hidden inputs to identify which form was submitted & accordingly execute the associated logic.
Even though hidden inputs are invisible to the user, it is actually very visible in the page's source. That means other users can still see and change the values you set in the hidden input, therefore we should not store any sensitive information in this field.

10.0 Message Flashing and Form Validations

Message flashing displays a message on the client after a certain action has taken place - the message then disappears the next time the page is requested. Flash messages exist for one redirect cycle, whereas session stays until it’s manually popped. We can use message flashing in combination with form validations. To use message flashing we need to do the following:
  • Import flash, session from flask. Make sure to setup the secret key for the app.
  • We can now involve the flash() function and pass it a string message we want to display on the client.
  • We also get access to the get_flashed_messages() function on the client side which we can use to get all flash messages as a list.

11.0 Responding with JSON 

When an API is communicated over the internet, following the rules of HTTP, we call it a RESTful API. One of the popular ways to send data using RESTful API is JSON which uses key-value pairs delimited by a colon. Lets update the db_setup.py and add a decorator method within the MenuItem class which will help define what data we want to send across and in a format that Flask can easily use.
#Insert this in the MenuItem class in db_setup.py
@property
def serialize(self):
return{
'name': self.name,
'description': self.description,
'id':self.id,
'price': self.price,
'course': self.course,
}
Flask has a builtin package called jsonify that will allow us to configure an API endpoint for our application. In this example, we will only focus on GET requests to get a list of menu items.
#webserver.py file
from flask import jsonify
#Make an API endpoint (GET request) to get a list of menu items
@app.route('/restaurants/<int:restaurant_id>/menu/JSON')
def restaurantMenuJSON(restaurant_id):
restaurant = session.query(Restaurant).filter_by(id=restaurant_id).one()
items = session.query(MenuItem).filter_by(restaurant_id=restaurant_id).all()
return jsonify(MenuItems = [i.serialize for i in items])

13.0 Encryption

We will need to encrypt user’s passwords before we store them in our database. Encryption masks the value of an input by using an algorithm that changes its value. We will use BCrypt to has the passwords. You need to install it using pip install flask-crypt
​​To use BCrypt, we need to do the following:
  • Import BCrypt, from flask_bcrypt import Bcrypt
  • Pass the application object to it, bcrypt = Bcrypt(app)
  • Generate a password hash (when storing): bcrypt.generate_password_hash(request.form[‘password’] - this will be the encrypted password we need to store in our database when creating a user during registration.
  • Compare password during logins: bcrypt.check_password_hash(user.password, request.form[‘password’]) , where user.password is the one retrieved from the database. We can use this in an if conditon to check if the password was a match or not.
Note: When using BCrypt, make sure that the password field in your database is 255 characters long - Bcrypt hashes are quite long, so this prevents any errors from the hashed password being truncated. 

14.0 Login & Registration - Complete Example

#db_setup.py
import sys
from sqlalchemy import Column, ForeignKey, Integer, String
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import relationship

Base = declarative_base()
class User(Base):
__tablename__ = 'user'
first = Column(String(15),nullable=False)
last= Column(String(15),nullable=False)
email = Column(String(80),nullable=False, unique=True)
password = Column(String(255),nullable=False)
id = Column(Integer, primary_key=True)
engine = create_engine('sqlite:///users.db')
Base.metadata.create_all(engine)
#server.py
from flask import Flask,redirect,request,render_template,url_for, flash
from flask import session as login_session
from flask_bcrypt import Bcrypt
import re
EMAIL_REGEX = re.compile(r'^[a-zA-Z0-9\.\+_-]+@[a-zA-Z0-9\._-]+\.[a-zA-Z]*$')
app = Flask(__name__)
bcrypt = Bcrypt(app)
app.secret_key = 'This is a secret key'
# Connect to database
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker
from db_setup import Base,User
engine = create_engine('sqlite:///users.db')
Base.metadata.bind = engine
DBSession = sessionmaker(bind=engine)
session = DBSession()
@app.route('/')
def index():
return render_template('index.html')
@app.route('/process' , methods = ['POST'])
def process():
if request.form['action'] == 'register':
if len(request.form['first'])<1 or len(request.form['last']) <1 or len(request.form['email']) <1:
flash('Field cannot be blank')
return redirect('/')
elif len(request.form['password'])<8:
flash('Password must be minimum 8 characters.')
return redirect('/')
elif not EMAIL_REGEX.match(request.form['email']):
flash('Not a valid email address')
return redirect('/')
elif request.form['password'] != request.form['cpassword']:
flash("Password's don't match.")
return redirect('/')
elif session.query(User).filter_by(email=request.form['email']).first():
flash('Email already registered. Select a different email address.')
return redirect('/')
else:
#Encrypt password received from the form
first = request.form['first']
last = request.form['last']
email = request.form['email']
password = bcrypt.generate_password_hash(request.form['password'])
#Create a user with the encrypted password
user = User(first=first,last=last,email=email,password=password)
session.add(user)
session.commit()
#Direct the user to the welcome page
login_session['user'] = email
print(login_session['user'])
return render_template('welcome.html',first=first,last=last)
elif request.form['action'] == 'login':
if len(request.form['email']) <1:
flash('Field cannot be blank')
return redirect('/')
elif not EMAIL_REGEX.match(request.form['email']):
flash('Not a valid email address.')
return redirect('/')
else:
user = session.query(User).filter_by(email=request.form['email']).one()
login_session['user'] = user.email
return render_template('welcome.html',first=user.first,last=user.last)
elif request.form['action'] == 'logout':
login_session.pop('user')
return redirect('/')
app.run(debug=True)
